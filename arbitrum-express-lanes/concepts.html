Four ideas worth knowing cold

Side-by-side comparisons instead of textbook definitions — the fastest way to actually remember which side is which.

Web2 vs Web3

who's in control

Web2

  • A company's server holds your data and account
  • You log in with a password the platform controls
  • The platform can change rules, fees, or bans unilaterally
  • Fast, familiar, but you're a guest on someone else's system

Web3

  • A blockchain network holds the record; no single server
  • You "log in" with a private key you alone hold
  • Rules live in open code that's hard to change unilaterally
  • Slower to evolve, but you own the underlying asset directly

Ethereum vs Bitcoin

what they're for

Bitcoin

  • Built for one job: a decentralized store of value / currency
  • Scripting is deliberately limited — simplicity is the feature
  • Fixed 21M supply, issuance halves roughly every 4 years
  • Security track record is its main selling point

Ethereum

  • A general-purpose computer — runs arbitrary smart contracts
  • Full programming environment (the EVM) on top of the ledger
  • Powers DeFi, NFTs, DAOs, and L2s like Arbitrum
  • Flexibility is the feature; also a larger attack surface

Public Key vs Private Key

the keypair

Public key

  • Derived from your private key, safe to share openly
  • Your wallet address is built from it
  • Lets anyone send you funds or verify your signature
  • Sharing it can't move funds out of your wallet

Private key

  • A secret number only you should ever know
  • Signs transactions to prove you authorized them
  • Whoever holds it fully controls the wallet — no recovery
  • Never enter it into a website; that's the #1 scam vector

Blockchain vs Traditional DB

how data is kept

Traditional database

  • One party (or team) owns and can edit the data directly
  • Fast writes, easy to update or delete rows
  • You trust the operator to keep it accurate and online
  • Great when a trusted party already exists

Blockchain

  • Thousands of independent nodes each hold a full copy
  • Appending is easy; rewriting history is intentionally hard
  • Trust comes from consensus rules, not one operator
  • Great when no single party should be trusted alone
